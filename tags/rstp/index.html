RSTP原理与报文详解

RSTP 知识体系分为四层：端口角色 → 端口状态 → BPDU Flags → P/A 与 TC。角色是"任命"，状态是"现在在干嘛"，Flags 是驱动状态转换的信号。 第一层：端口角色（Port Role）—— 端口在拓扑中的身份 选举结束后，每个端口被分配一个角色： 角色 英文 谁拥有 朝根方向 转发数据 根端口 Root Port（RP） 每台非根桥有且仅 1 个 ↑ 朝根 ✅ 指定端口 Designated Port（DP） 每条网段有且仅 1 个 ↓ 背根 ✅ 替代端口 Alternate Port（AP） 下游备用端口 ↑ 朝根但封住 ❌ 备份端口 Backup Port（BP） 同一网段备用端口 — ❌ 边缘端口 Edge Port（EP） 连终端/服务器，管理员配置 ↓ ✅ 判定逻辑： RP：凡是通往 RB 的通路都是 RP DP：凡是下游有其他交换机的都是 DP AP：凡是通往 RB 但是被阻断的都是 AP BP：凡是交换机自己连自己的都是 BP EP：连接终端且管理员配置的可以成为 EP AP vs BP 的区分：去掉 RP 和 DP 后，剩下能收 BPDU 的端口——收到别人发出的 BPDU 是 AP，收到自己发出的 BPDU 是 BP。收到自己的报文说明存在自身环路或集线器，前者是错误，后者已被淘汰，因此备份端口实际上很罕见。 ...
